# Trap-PR-Agent

Trap-PR-Agentは未来を生きるプログラマーを鍛えるアプリです。
AIからのプルリクに含まれる罠を見破り、コードを見る目を磨きましょう！

## 🚀 使い方

### 1. 初期設定（ユーザー名とトークンの入力）
最初の画面でGitHubのユーザー名とGitHubパーソナルアクセストークンを入力します。

<img width="928" height="532" alt="image" src="https://github.com/user-attachments/assets/a51ef156-bbc1-4bdc-95f0-e7a91ee1f372" />

#### 🔑 GitHubパーソナルアクセストークンの発行手順
1. 左メニューの一番下 **Developer settings** > **Personal access tokens** > **Tokens (classic)** をクリック。
2. **Generate new token (classic)** をクリック（パスワードを求められる場合があります）。
3. **Note** に `Trap-PR-Agent-Token` など適当な名前をつける。
4. **Select scopes** で、`repo`（リポジトリの全権限）にチェックを入れる。
5. 一番下の **Generate token** をクリックし、出てきた `ghp_` から始まる文字列をコピーする。（※この画面を閉じると二度と見られないので注意！）

### 2. 罠PRの作成と挑戦
- 右下の「**罠PRを作成**」ボタンを押し、PRを作成したいリポジトリとパス（任意）を選択・入力します。
- 無事に罠PRが生成されたら「**GitHubで開く**」を押すと、生成された実際の罠PRに飛びます。
- 変更点を確認し、PRに潜む罠をコメントで指摘するとAIによって採点され記録されます。

### 3. 履歴とプロフィールの確認
- 「**履歴**」タブでは過去に挑戦した罠PRを一覧で確認できます。
- 「**プロフィール**」タブではこれまでの挑戦回数や分野ごとの得点率が確認できます。

## 1. 本作品で解決したい課題とその背景

近年、多くの開発現場では品質向上やトラブル防止のために、GitHubのプルリクエスト等においてコードレビューが行われています。しかしレビュー経験の少ない学生や初学者は、「どこを見ればよいのか」「自分のレビューが適切なのか」を判断することが難しく、実践的に学ぶ機会も限られています。

また、コードレビューは実際の開発経験を通して習得することが多く、初心者が繰り返し練習できる環境は十分に整っていません。

そこで、実際のGitHub Pull Requestを教材として利用し、AIによる採点とフィードバックを受けながらレビューを学べる環境を提供することで、効率的にコードレビュー能力を向上させることを目指しました。

## 2. 想定する利用ユーザー

- プログラミング初学者
- 大学生・専門学生
- GitHubを使い始めた開発者
- コードレビュー経験が少ないエンジニア
- コードレビュー能力を向上させたい開発者

## 3. プロダクトの特徴

### ① 実際のGitHub Pull Requestを教材として利用
実際のPull Requestをレビュー対象とすることで、実務に近い環境でレビューを学習できます。

### ② AIによるレビュー採点
Geminiがレビュー内容を分析し、スコアや発見できた問題、見落とした問題、改善点をフィードバックします。

### ③ AIチャットによる学習支援
レビュー中に分からない内容をAIへ質問でき、その場で疑問を解決しながら学習できます。

### ④ 学習履歴の管理
過去のレビュー結果やスコアを確認でき、自身の成長を振り返ることができます。

### ⑤ スキルの可視化
プロフィール画面ではレビュー実績や正答率等を確認できます。
